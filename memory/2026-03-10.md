## Dashboard Table Layout — Evening Session (v1.6+)
Long iteration on Open Positions table layout. Key decisions made:

### Problem
At 1920px desktop, Open Positions table had:
1. Dead space gap between title text and Signal column
2. Title truncating at "Mar 10, 2" (missing "2026")
3. Right columns (Signal/Edge/Qty/Side/Ent/Cost/P&L) cramped

### Root causes identified
- `table-layout: fixed` with colgroup % widths caused proportional dead space
- `max-width: 0` + `width: 100%` on title th forced title column to take all space, leaving 65% empty cell
- `-webkit-line-clamp: 2` on `<a>` conflicted with auto width
- BOT badge on "line 2" below title text — "2026" would be line 2 of title but badge displaced it

### Final approach (patch_natural_title.py)
- `table-layout: auto` (switched from fixed)
- Removed colgroup entirely
- All non-title columns: `width: 1%; white-space: nowrap` (shrink to content)
- Title th: NO explicit width (natural auto distribution)
- Title td: NO max-width
- `<a>` in title: `flex: 0 1 auto; max-width: 420px` — takes content width, does NOT stretch
- BOT/MANUAL badge moved to LEFT of title text (inline flexbox row: [badge][title...])
- Removed CUR column — reduces Open Positions from 9 → 8 columns
- Page max-width: 1440px → 1920px (fills full desktop screen)

### Current column order (Open Positions, 8 cols)
POSITION | SIGNAL | EDGE | QTY | SIDE | ENT | COST | OPEN P&L

### CSS classes active
- `.tbl { table-layout: auto; }`
- `.shrink { width: 1%; white-space: nowrap; }`
- `.ra { text-align: right; }`, `.ca { text-align: center; }`
- `.col-title` flex div: `display:flex; align-items:center; gap:6px`
- Signal th: `min-width: 80px`, Edge th: `min-width: 45px`
