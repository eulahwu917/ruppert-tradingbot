### Architecture Decision: Trading Strategy Layer Separation
- **Decision**: All entry/exit/sizing logic moves to a separate overarching Trading Strategy layer
- Market modules (Weather, Crypto) become pure data providers — return only: `{prob, confidence, edge, direction}`
- Trading Strategy layer handles all capital deployment decisions — modules don't know $25 exists
- Benefit: adding Jobs/Commodities later = zero changes to trading logic

### Position Sizing Finalized
- Max per ticker: $50 (raised from $25 initial entry; $25 entry + up to $25 add-on)
- Per-trade entry: `min($25, 2.5% of capital)` Kelly-weighted
- Daily cap: 70% of total capital (shared pool across all modules); 30% always reserved
- Add-on scale (Bayesian proportional to confidence delta):
  - drift +10–25% → add 25% of remaining allocation
  - drift +25–50% → add 50% of remaining allocation
  - drift +50%+   → add 100% (fill to max)

### Full Trade Flow (4 phases confirmed with David)
1. **Identify**: Module scan → signal/edge/confidence → flag if edge ≥ threshold
2. **Entry**: Kelly sizing → cap check → execute → log
3. **Positioning**: re-evaluate each scan cycle; add if signal strengthens, trim/exit if reverses
4. **Exit**: settle OR sell early per take-profit/stop rules

### Exit Rules (full ladder)
- **95¢ rule (PRIORITY)**: If `no_bid ≥ 95¢` (or yes_bid ≥ 95¢) → sell immediately; capital efficiency beats 1¢ remaining gain
- **70% gain rule**: If position captured ≥70% of max profit → sell; risk/reward flips
- **Signal reversal stop-loss**:
  - Reversed 10–20%: trim 25%
  - Reversed 20–35%: sell 50%
  - Reversed 35%+: exit 100% immediately
- **Settlement**: <30 min to expiry → hold, do not exit (fees + spread not worth it)

### Quant Research Learnings Applied
- **Fractional Kelly at 25%**: confirmed optimal range for prediction markets (20–30%)
- **Anti-Martingale confirmed**: add to winners, reduce losers — superior to flat sizing
- **Bayesian updating** (post-Friday): replace raw drift thresholds with Bayesian probability updates — accounts for prior confidence at entry
- **Volatility-adjusted sizing for crypto**: `Kelly_adj = standard_Kelly × (baseline_vol / current_vol)` — high vol → smaller position
- **ECMWF upgrade** (post-Friday): add ECMWF ensemble alongside GFS; more accurate for 1–5 day temps

### Weather Module Review
- Core GFS 31-member ensemble approach: solid
- **T-market NO-only rule REMOVED**: was forcing NO regardless of signal — wrong
- **Replacement**: soft confidence prior for T-markets:
  - Signal says NO on T-market → confidence × 1.15 (longshot bias boost)
  - Signal says YES on T-market → confidence × 0.85 (reduce)
  - If signal >70% prob either way → adjustment ignored, signal wins
- **Known issues to fix**: Miami NWS grid (MFL 110,37) 404; bias corrections (+2-4°F) based on 1-day backtest only; same-day 60/40 blend is arbitrary

### Crypto Module Review
- Core signal: 6-factor momentum score (RSI, MA, 1h/4h/24h/7d change) → BULLISH/BEARISH/NEUTRAL
- Probability model: Normal distribution (mu = current_price + momentum_shift; sigma = price × hourly_vol × √hours)
- Smart money: 3 Polymarket wallets → boost confidence medium→high if aligned
- **Critical gap**: Normal distribution is WRONG for crypto — fat tails and skew; log-normal is more accurate
- **Other gaps**: Only 3 wallets tracked (thin); momentum is binary (up/down, no magnitude); 8% momentum shift arbitrary; confidence categorical not numeric
