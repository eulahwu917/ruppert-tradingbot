## Check 3: Correctness Traces — ✅ PASS

### Trace A: `_spread = -5` → `_spread_score`

```
1.0 - (-5 / 20.0)
= 1.0 - (-0.25)
= 1.25
```
Without the fix: `max(0.0, 1.25)` → **1.25** ❌ (exceeds 1.0)
With the fix:    `min(1.0, max(0.0, 1.25))` → **1.0** ✅

Verdict: Correctly clamped to 1.0 for inverted spreads. Fix is mathematically sound.

### Trace B: Two Chicago markets pass edge — only higher-edge survives dedup

Scenario:
- Opp A: `city=Chicago`, `target_date=2026-03-27`, `edge=0.18`, `ticker=KXCHI-T44`
- Opp B: `city=Chicago`, `target_date=2026-03-27`, `edge=0.12`, `ticker=KXCHI-B41.5`

After sort (descending edge): [Opp A (0.18), Opp B (0.12)]

Dedup iteration:
1. Opp A: key=`(Chicago, 2026-03-27)` not seen → added to `_seen_city_date`, appended to `_deduped_opps`
2. Opp B: key=`(Chicago, 2026-03-27)` already seen → logged as DEDUP skip, NOT appended

Result: Only Opp A (edge=0.18) survives. ✅
Log emitted: `[Weather] DEDUP skip KXCHI-B41.5 — city=Chicago date=2026-03-27 already covered by KXCHI-T44`
