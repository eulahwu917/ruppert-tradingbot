## Background

Kalshi has 4 live 15-min binary crypto direction series:
- `KXBTC15M` — "Will BTC price be up in the next 15 mins?"
- `KXETH15M` — ETH equivalent
- `KXXRP15M` — XRP equivalent
- `KXDOGE15M` — DOGE equivalent

These are binary YES/NO markets. Settlement: YES if price at close > price at window open (Coinbase reference price). These are structurally different from the hourly band markets — no strike, no band, just direction.

The existing `evaluate_crypto_entry()` in `position_monitor.py` does NOT handle these — it looks for B/T strike prefixes and silently returns on 15-min tickers. A new function is required.

### Signal Stack

#### Signal 1: Taker Flow Imbalance (TFI) — weight 0.40
**Source:** Binance Futures REST — `GET /fapi/v1/takerBuySellVol?symbol=BTCUSDT&period=5m&limit=3`

```python
# Per 5-min bucket:
TFI_t = (buyVol - sellVol) / (buyVol + sellVol)  # ∈ [-1, 1]

# Time-weighted (most recent bucket counts most):
weights = [0.20, 0.30, 0.50]  # t-10min, t-5min, t-0min
TFI_composite = sum(w * tfi for w, tfi in zip(weights, buckets))

# Z-score against 4H rolling window (48 × 5-min buckets):
TFI_z = (TFI_composite - mean_4h) / std_4h
TFI_z = clip(TFI_z, -3, 3)
```

Mark stale if last update > 90s old → block entry.

#### Signal 2: Orderbook Imbalance (OBI) — weight 0.25
**Source:** Binance Futures REST — `GET /fapi/v1/depth?symbol=BTCUSDT&limit=20`

```python
# Top-10 levels:
OBI_instant = (sum(bid_qty[:10]) - sum(ask_qty[:10])) / (sum(bid_qty[:10]) + sum(ask_qty[:10]))

# EWM over recent snapshots (target half-life ~60s):
OBI_ewm = ewm(OBI_snapshots, span=60)

# Z-score (4H rolling):
OBI_z = (OBI_ewm - mean_4h) / std_4h
OBI_z = clip(OBI_z, -3, 3)
```

Mark stale if last snapshot > 30s old → block entry.

#### Signal 3: MACD Histogram on 5-min candles — weight 0.15
**Source:** Binance Futures REST — `GET /fapi/v1/klines?symbol=BTCUSDT&interval=5m&limit=30`

```python
EMA_12 = ewm(closes, span=12)
EMA_26 = ewm(closes, span=26)
MACD_line = EMA_12 - EMA_26
MACD_signal = ewm(MACD_line, span=9)
MACD_hist = MACD_line - MACD_signal

# Z-score (4H rolling):
MACD_z = (MACD_hist - mean_4h) / std_4h
MACD_z = clip(MACD_z, -3, 3)
```

Do NOT use hourly RSI — wrong timescale, do not port from crypto_client.py.

#### Signal 4: Open Interest Delta — weight 0.10
**Source:** Binance Futures REST — `GET /futures/data/openInterestHist?symbol=BTCUSDT&period=5m&limit=3`

```python
OI_delta_pct = (OI_t - OI_{t-5min}) / OI_{t-5min}
price_delta_5m = (price_t - price_{t-5min}) / price_{t-5min}

# Conviction: rising OI in same direction as price = strong; falling OI = weak
OI_conviction_raw = OI_delta_pct * sign(price_delta_5m)

# Z-score (4H rolling):
OI_conviction_z = (OI_conviction_raw - mean_4h) / std_4h
OI_conviction_z = clip(OI_conviction_z, -2, 2)
```

#### Bias Filters (not in main score)
- **Funding rate z-score** (already in `crypto_client.py`): if FR_z > 2.0, multiply P by 0.85; if FR_z < -2.0, multiply by 1.15
- **Polymarket divergence** (if available): if |polymarket_yes - kalshi_yes| > 0.03, add `0.3 * divergence` to P (additive nudge only)

### Composite Score → Probability

```python
raw_score = (0.40 * TFI_z + 0.25 * OBI_z + 0.15 * MACD_z + 0.10 * OI_conviction_z)

# Sigmoid transform — scale factor starts at 1.0, will be calibrated from data
SCALE = getattr(config, 'CRYPTO_15M_SIGMOID_SCALE', 1.0)
P_directional = 1 / (1 + exp(-SCALE * raw_score))

# Apply bias filters
P_biased = P_directional * funding_multiplier
P_final = clip(P_biased + poly_nudge, 0.05, 0.95)
```

Add `CRYPTO_15M_SIGMOID_SCALE = 1.0` to `config.py`. Autoresearcher will tune this over time.

### Timing Gate

```python
elapsed_secs = (now - window_open_time).total_seconds()

if elapsed_secs < 120:
    return NO_ENTRY  # price discovery chaos, skip

elif elapsed_secs <= 480:
    min_edge = config.CRYPTO_15M_MIN_EDGE  # default 0.08

elif elapsed_secs <= 660:
    min_edge = config.CRYPTO_15M_MIN_EDGE * 1.25  # secondary window, harder threshold
```

### Entry Logic

```python
edge_yes = P_final - (yes_ask / 100)
edge_no  = (1 - P_final) - (no_ask / 100)

if edge_yes >= min_edge:
    direction = 'yes'
    entry_price = yes_ask
    P_win = P_final

elif edge_no >= min_edge:
    direction = 'no'
    entry_price = 100 - yes_bid  # no_ask
    P_win = 1 - P_final

else:
    return NO_ENTRY
```

### Position Sizing

Half-Kelly, capped:
```python
c = entry_price / 100
kelly_full = (P_win - c) / (c * (1 - c))
kelly_half = kelly_full / 2

position_usd = min(
    kelly_half * capital,
    capital * config.MAX_POSITION_PCT,   # 1% of capital
    100.0                                 # hard $100 cap
)
position_usd = max(position_usd, 5.0)  # minimum viable
```

### Risk Filters (hard blocks — any True = no entry)

```python
# R1: Extreme realized vol (>3x 30d avg)
if vol_5min > 3.0 * rolling_30d_avg_vol: block("EXTREME_VOL")

# R2: Kalshi spread too wide
if (yes_ask - yes_bid) > 8: block("WIDE_SPREAD")

# R3: Kalshi book too thin
if book_depth_usd < 1000: block("LOW_KALSHI_LIQUIDITY")

# R4: Binance volume anomaly (thin underlying)
if binance_5min_vol < 0.25 * rolling_30d_avg_binance_vol: block("THIN_MARKET")

# R5: Stale data
if tfi_age > 90: block("TFI_STALE")
if obi_age > 30: block("OBI_STALE")

# R6: Extreme funding
if abs(fr_z) > 3.0: block("EXTREME_FUNDING")

# R7: Weak/conflicting signals
if abs(raw_score) < 0.15: block("LOW_CONVICTION")

# R8: Session drawdown
if session_pnl < -0.05 * daily_15m_allocation: block("DRAWDOWN_PAUSE")

# R9: Macro event within 30 min (reuse existing context from main cycle)
if has_macro_event_within(minutes=30): block("MACRO_EVENT_RISK")

# R10: Coinbase-Binance basis too wide (settlement risk)
# Fetch Coinbase BTC price: GET https://api.coinbase.com/v2/prices/BTC-USD/spot
if abs(coinbase_price - binance_price) / binance_price > 0.0015: block("BASIS_RISK")
```
