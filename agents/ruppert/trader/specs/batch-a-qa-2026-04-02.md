## Fix 2: Band daily sigma per-market

**Files changed:** `agents/ruppert/trader/crypto_band_daily.py`

**What to verify:**
- `_hours_left` is now computed **before** `prob_model` (moved earlier in the market loop)
- `sigma_m = daily_vol * math.sqrt(max(1.0, _hours_left) / 24.0)` is computed per-market
- `band_prob()` is called with `sigma_m` (not the outer `sigma` from `SERIES_CFG`)
- The outer `sigma = daily_vol * math.sqrt(hours / 24)` still exists in `SERIES_CFG` loop (used for `drift`)
- `new_crypto.append(...)` stores `'sigma': sigma_m` (the per-market value)
- `_log_band_decision()` SKIP calls for `edge_below_threshold` and `price_too_high` also log `sigma_m`

**Test approach:**
```python
import math
from agents.ruppert.trader.crypto_band_daily import band_prob

# Simulate two contracts: same-day (~6h left) vs next-day (~30h left)
spot = 70000.0
band_mid = 70000.0
half_w = 250.0
daily_vol = 0.025
drift = 0.0

sigma_6h  = daily_vol * math.sqrt(6.0 / 24.0)
sigma_30h = daily_vol * math.sqrt(30.0 / 24.0)

prob_6h  = band_prob(spot, band_mid, half_w, sigma_6h, drift)
prob_30h = band_prob(spot, band_mid, half_w, sigma_30h, drift)

# Next-day contract (30h) should have LOWER probability of staying in band
# (wider distribution = more likely to leave the band)
assert prob_30h < prob_6h, f"Expected prob_30h({prob_30h:.4f}) < prob_6h({prob_6h:.4f})"
print(f"PASS: prob_6h={prob_6h:.4f}, prob_30h={prob_30h:.4f} (sigma_6h={sigma_6h:.4f}, sigma_30h={sigma_30h:.4f})")
```

**Acceptance:** `prob_30h < prob_6h` — next-day contracts use larger sigma, correctly reflecting longer horizon uncertainty.

## Fix 3: Strike distance gate in threshold_daily

**Files changed:**
- `agents/ruppert/trader/crypto_threshold_daily.py` — `select_best_strike()` signature + gate logic
- `environments/demo/config.py` — `CRYPTO_1D_MAX_STRIKE_DISTANCE_PCT = 0.02`

**What to verify:**
- `select_best_strike()` accepts `current_price: float = 0.0` parameter
- Strike is parsed from ticker via `float(ticker.split('-T')[-1])`
- Strikes where `abs(strike - current_price) / current_price > 0.02` are skipped with a debug log
- Strikes within 2% of spot are still evaluated normally
- When `current_price=0.0` (default), the gate is bypassed (no skip)
- `evaluate_crypto_1d_entry()` passes `current_price=current_price` to `select_best_strike()`
- `config.CRYPTO_1D_MAX_STRIKE_DISTANCE_PCT` exists and equals `0.02`

**Test approach:**
```python
import config
from agents.ruppert.trader.crypto_threshold_daily import select_best_strike

# Verify config key exists
assert hasattr(config, 'CRYPTO_1D_MAX_STRIKE_DISTANCE_PCT'), "Config key missing"
assert config.CRYPTO_1D_MAX_STRIKE_DISTANCE_PCT == 0.02

# Spot = 70000; 2% gate = max distance $1400
spot = 70000.0

# Within range: strike at $70500 (0.71% away) — should be evaluated
# Out of range: strike at $73000 (4.3% away) — should be skipped
markets_in  = [{'ticker': 'KXBTCD-T70500', 'yes_ask': 50, 'no_ask': 52}]
markets_out = [{'ticker': 'KXBTCD-T73000', 'yes_ask': 10, 'no_ask': 92}]
markets_both = markets_in + markets_out

# With current_price=0 (default) — gate bypassed
result_no_gate = select_best_strike('BTC', 0.6, markets_both, direction='above', current_price=0.0)
# Both evaluated; out-of-range may win on edge (low ask)
print(f"No gate: picked {result_no_gate.get('ticker') if result_no_gate else None}")

# With current_price=70000 — gate active; far strike skipped
result_gated = select_best_strike('BTC', 0.55, markets_both, direction='above', current_price=spot)
if result_gated:
    picked_ticker = result_gated['ticker']
    assert 'T70500' in picked_ticker, f"Gate failed: picked far strike {picked_ticker}"
    print(f"PASS (gated): selected near-strike {picked_ticker}")
else:
    print("No qualifying strike (edge < min_edge) — adjust P_above or market ask for test")

# Out-of-range only — should return None
result_far_only = select_best_strike('BTC', 0.55, markets_out, direction='above', current_price=spot)
assert result_far_only is None, "Expected None for all-far-strike market list"
print("PASS: all-far-strike market list correctly returns None")
```

**Acceptance:**
1. `CRYPTO_1D_MAX_STRIKE_DISTANCE_PCT = 0.02` in config
2. Strikes >2% from spot are skipped when `current_price > 0`
3. Within-range strikes still selected normally
4. When `current_price=0`, gate is bypassed
