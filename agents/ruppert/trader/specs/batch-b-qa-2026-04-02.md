## Summary

## Fix 1: Decisions Logging

### Files changed
- `agents/ruppert/data_analyst/ws_feed.py` — `evaluate_crypto_entry()`
- `agents/ruppert/trader/crypto_threshold_daily.py` — `_log_decision()` signature

### What changed

**ws_feed.py:**
- Added `_log_threshold_skip()`, `_log_band_skip()`, and `_log_skip()` helpers inside `evaluate_crypto_entry()`.
- SKIP cases now logged: `daily_cap_reached`, `circuit_breaker`, `strategy_gate:<reason>`, `size_below_minimum`
- ENTER case: after trade is logged and position tracked, calls `_log_decision` (threshold) or `_log_band_decision` (band) with full fields.

**crypto_threshold_daily.py `_log_decision()`:**
- Added optional params: `ticker`, `side`, `model_prob`, `end_date`
- `end_date` written to entry only when provided (no schema break)
- `ticker` and `side` included in every entry now

## Fix 2: CB Cleanup Finalizer

### Files changed
- `environments/demo/scripts/clean_break.py`
- `environments/demo/scripts/pnl_correction.py`
- `environments/demo/scripts/migrate_module_taxonomy.py`
- `environments/demo/scripts/pnl_correction_module_id.py`

### What changed
Added CB state refresh block at the end of each mutation script:
```python
try:
    import agents.ruppert.trader.circuit_breaker as _cb
    from agents.ruppert.data_scientist.capital import get_capital as _get_capital
    _cb.update_global_state(_get_capital())
    print('[CB] Global state refreshed after cleanup.')
except Exception as e:
    print(f'[CB] State refresh failed (non-fatal): {e}')
```
